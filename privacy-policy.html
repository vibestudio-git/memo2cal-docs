1. Information we collect and how (no developer servers)

  • We do not operate developer servers, nor do we transmit/store personal data on any developer-operated backend.
  • Google account auth info: OAuth tokens for Google Calendar API access (stored locally)
  • Calendar data access: list/create/update/delete events via Google APIs with explicit user consent only
  • User inputs: memo text, reminder interval, working hours, and other settings
  • Technical info: minimal diagnostics (errors) stored locally

2. Purposes of processing

  • Create Google Calendar events from quick memos
  • Provide periodic reminders to build a memo habit
  • Store/apply user preferences (time window, reminder interval)
  • Improve stability with minimal diagnostics

3. Storage and security

  • Local storage only (Chrome storage on the user’s device)
  • HTTPS for all communications with Google APIs
  • Least-privilege permissions, used strictly for features

4. Third‑party sharing and transfers

  • No transfer to developer servers; no selling/sharing with third parties.
  • Calendar data is sent only to Google via Google APIs within the scope of user consent. The Service does not proxy/store such data on developer servers.

5. Google policy compliance (“Limited Use”)

Our use and transfer of information received from Google APIs will adhere to the Google API Services User Data Policy, including the “Limited Use” requirements.

6. Your rights and choices

  • Access/Rectification/Deletion: manage events directly in Google Calendar. After uninstalling, local data is no longer used.
  • Revocation: disable the extension in Chrome, or revoke access in your Google Account (third‑party app access).

7. Retention and deletion

  • Data is kept locally only while the Service is in use.
  • Uninstalling removes local data on the device; some temporary data may be periodically cleared.

8. Cookies/analytics/tracking

  • No third‑party analytics or tracking tools
  • No cookies stored by this extension

9. Children’s privacy

Not directed to children under 13; we do not knowingly collect children’s personal information.

10. International transfers

No developer servers are used. Processing via Google APIs follows Google’s systems and policies.

11. Permissions and purposes (summary)

  • identity: Google OAuth and Calendar API access
  • storage: local storage of settings/temporary memo state
  • alarms: schedule periodic reminders
  • notifications: show memo reminders and actions
  • windows, tabs, system.display: open and place a popup when minimized/in background (accessibility)
  • host_permissions: communication with googleapis and calendar.google.com

12. Changes and notice

We will provide reasonable notice via the extension when this policy changes and update the “Last updated” date above.

1. 수집하는 정보와 수집 방법 (개발자 서버 미사용)

  • 개발자 운영 서버를 사용하지 않으며, 개인정보를 전송/저장하지 않습니다.
  • Google 계정 인증 정보: Google Calendar API 접근을 위한 OAuth 토큰(로컬 저장)
  • 캘린더 데이터 접근: 사용자 동의 하에 Google API로 일정 조회/생성/수정/삭제
  • 사용자 입력 데이터: 메모 내용, 알림 간격, 업무 시간대 등 설정
  • 기술적 정보: 오류 등 최소 진단 정보(로컬 보관)

2. 이용 목적

  • 빠른 메모를 Google 캘린더 일정으로 생성
  • 주기 알림 제공으로 메모 습관 형성
  • 사용자 설정(시간대/간격) 저장 및 적용
  • 안정성 개선을 위한 최소 진단

3. 저장 위치 및 보안

  • 로컬 저장(사용자 기기의 Chrome storage)
  • Google API 통신은 HTTPS 사용
  • 기능 제공에 필요한 최소 권한만 사용

4. 제3자 제공 및 외부 전송

  • 개발자 서버로 전송하지 않으며, 제3자에 판매/공유하지 않습니다.
  • 캘린더 데이터는 사용자 동의 범위 내에서만 Google API로 전송되며, 개발자 서버에 중계/저장하지 않습니다.

5. Google 정책 준수(“Limited Use”)

본 서비스는 Google API로부터 수신한 정보의 사용/전송에 관하여 Google API Services User Data Policy ("Limited Use" 요건 포함)를 준수합니다.

6. 이용자 권리와 선택

  • 열람/정정/삭제: Google 캘린더에서 직접 관리. 확장 프로그램 제거 시 로컬 데이터는 더 이상 사용되지 않습니다.
  • 철회: Chrome 확장프로그램 관리 또는 Google 계정(제3자 앱 접근)에서 접근 권한 해지 가능

7. 보유 및 파기

  • 서비스 이용 기간 동안에 한하여 로컬 데이터 사용
  • 제거 시 기기 내 로컬 데이터 삭제, 일부 임시 데이터는 주기적으로 초기화될 수 있음

8. 쿠키/분석/추적

  • 제3자 분석/추적 도구 미사용
  • 확장 프로그램 특성상 별도 쿠키 저장 없음

13세 미만 아동을 대상으로 하지 않으며, 아동의 개인정보를 의도적으로 수집하지 않습니다.

10. 국제 전송

개발자 서버를 사용하지 않으며, Google API 이용에 따른 처리는 Google의 시스템/정책을 따릅니다.

11. 권한과 사용 목적(요약)

  • identity: Google OAuth 및 Calendar API 접근
  • storage: 설정/임시 메모 상태 로컬 저장
  • alarms: 주기 알림 스케줄링

  • host_permissions: googleapis, calendar.google.com 통신

정책 변경 시 합리적인 방법으로 고지하고 상단의 “마지막 업데이트” 날짜를 갱신합니다.
